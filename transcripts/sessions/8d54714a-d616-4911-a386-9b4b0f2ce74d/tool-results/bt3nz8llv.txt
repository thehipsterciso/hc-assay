### 4. [HIGH] HIGH_STAKES tier has no inner timeout/cancellation — hung subprocess calls permanently leak the global pool's worker slots, killing reasoning process-wide for ALL concurrent studies
- **Location:** `src/assay_engine/reasoning/seam.py` 181-217, 361-439, 442-490
- **Problem:** The reasoning seam routes every LLM call through a single module-global, 8-worker ThreadPoolExecutor (_pool) shared across all threads and all concurrent run_study invocations. _with_timeout enforces only a WALL-CLOCK timeout via future.result(timeout=...); it explicitly does NOT cancel the worker (the docstring at lines 226-229 acknowledges a ThreadPoolExecutor cannot interrupt a thread already blocked in a backend call). The slot is only released by the add_done_callback (_release), i.e. when the underlying call actually returns. For the BULK tier this is survivable because _bulk_complete passes client_kwargs={'timeout': BULK_TIMEOUT} to ChatOllama, so the worker is guaranteed to return and free its slot. If the claude CLI subprocess hangs (stalled subscription endpoint, network blackhole, child process wedged), the worker thread blocks forever and its pool slot is NEVER released. _inflight stays elevated permanently.
